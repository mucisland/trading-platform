# Patterns and Lessons Learned

## Purpose

This file captures recurring patterns, failures, and best practices discovered during development.

It exists to:
- prevent repeated mistakes
- improve agent performance over time
- encode "Ralph warnings" into durable guidance

## How to use

Add entries when:
- a failure occurs more than once
- a bug reveals a systemic weakness
- a better pattern is discovered
- a rule needed to be clarified

Each entry should include:
- context
- failure mode
- root cause
- recommended pattern

## Format

### Pattern: <short name>

Context:
- where it occurred

Failure:
- what went wrong

Cause:
- why it happened

Solution:
- what to do instead

Rule:
- concise enforceable rule
